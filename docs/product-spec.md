# Product Specification

## Overview

This project aims to build a mobile-friendly web application for running small, in-house fencing tournaments. The application replaces manual paper-based systems with a fast, simple digital workflow while enabling basic performance tracking.

## Problem

### Current Issues

- Manual poule creation is time-consuming and error-prone
- Tournament setup disrupts training flow
- Physical materials are unreliable or unavailable
- Results are not retained for future analysis

### Impact

- Slower training sessions
- Friction during coaching
- Lost opportunities for performance tracking

## Target Users

- Coaches running in-house fencing tournaments (initially single user)

## Scope (MVP)

The system supports running a full tournament:
- Poules → Rankings → DE bracket → Final results

## Constraints

- Single-user system
- Mobile-first design (phone browser)
- Small tournaments (≤ 25 fencers)
- Minimal stored data
- No authentication
- Cloud-hosted backend

## Non-Goals

- No multi-user support
- No analytics dashboards
- No public sharing
- No real-time collaboration
